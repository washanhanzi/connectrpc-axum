## Takeaways

- Pure protobuf encode still favors beta.5/prost, especially for the large payload.
- Protobuf decode favors `buffa` across all tested sizes.
- JSON encode strongly favors `buffa`; JSON decode is mixed and mostly neutral outside the small case.
- End-to-end Connect unary and Connect streaming paths favor `buffa` in this setup.
- Build/codegen timing is close enough that one run should be treated as directional, not conclusive.

## Caveats

- The beta.5 side uses the published crates.io release rather than the local `target/package` tree because Cargo cannot keep both local path sources in one lockfile when the package name and version are identical.
- The streaming benchmark results above come from the rerun after the harness started redirecting `stderr` to `/dev/null`; the earlier run was distorted by per-message debug output from the runtime.
- No client-side benchmark phase is included in this first report.
