# Fortune Benchmark: 2026-03-26

## Environment

- Host: `Linux ravenz 6.19.6-arch1-1 x86_64`
- CPU: `AMD RYZEN AI MAX+ PRO 395 w/ Radeon 8060S`
- `rustc`: `1.94.0 (4a4ef493e 2026-03-02)`
- `cargo`: `1.94.0 (85eff7c80 2026-01-15)`

## Command

```bash
VALKEY_ADDR=127.0.0.1:6389 cargo bench \
  --manifest-path benchmarks/compare-buffa-beta5/Cargo.toml \
  -p compare-buffa-beta5-runner \
  --bench fortunes
```

`VALKEY_ADDR` pointed at a manually started local `valkey/valkey:8-alpine` container so the run would avoid a Docker ephemeral-port allocation race. The benchmark path is otherwise the normal external-Valkey flow.

## Results

Warmup: `3.0s`, measurement: `10.0s`, mode: `default`

| Implementation | Concurrency | req/s | p50 (us) | p99 (us) |
|---|---:|---:|---:|---:|
| buffa | 16 | 143184 | 94 | 330 |
| v0.1.0 | 16 | 142331 | 104 | 302 |
| connect-rust | 16 | 127797 | 116 | 328 |
| buffa | 64 | 223593 | 267 | 547 |
| v0.1.0 | 64 | 222815 | 275 | 606 |
| connect-rust | 64 | 157206 | 320 | 1587 |
| buffa | 256 | 164290 | 1163 | 7482 |
| v0.1.0 | 256 | 162207 | 1156 | 7030 |
| connect-rust | 256 | 57677 | 2288 | 17799 |

## Read

- `buffa` and `v0.1.0` are effectively tied on this Fortune workload, with `buffa` slightly ahead on throughput at all three concurrency levels.
- `connect-rust` is slower on this harness at every tested concurrency level, and the gap widens materially at `64` and `256`.
- Tail latency is close between `buffa` and `v0.1.0` at `16` and `64`, while both degrade at `256`; `connect-rust` tails degrade much more sharply at higher concurrency in this setup.
